I have a particular problem that has to do with curriculum implementation in a college class that uses instructure canvas. My students do a weekly project during the semester and each project has an assignment page on canvas. On that page are the requirements, etc, but more importantly, the page contains the links to the two lectures I gave that week on YouTube. These are live streamed the week of the class. Is there a way to set these links well before the semester begins, then just update what they're pointing at after the lectures are completed? I want to avoid copying and pasting from YouTube every week if possible. Edit You're dealing with a common workflow challenge in Canvas, and there are several approaches that can streamline this process significantly. Best Solution: Use Consistent URL Structure with Redirects The most elegant approach is to create your own redirect system: Set up a simple redirect page on your own website/domain (or use a URL shortener service that allows editing) Create consistent URLs like: yoursite.com/week1-lecture1 yoursite.com/week1-lecture2 yoursite.com/week2-lecture1 etc. Add these permanent links to Canvas at the beginning of the semester Update the redirects each week to point to the actual YouTube URLs after streaming Alternative Canvas-Native Solutions: Option 1: Canvas External Tools/LTI If your institution supports it, you could embed a simple custom LTI tool that dynamically pulls YouTube URLs from a spreadsheet or database you update. Option 2: Canvas API Automation Write a script using Canvas's REST API to automatically update assignment pages. You'd maintain a spreadsheet with YouTube URLs and run the script weekly to update all relevant pages. Option 3: Placeholder YouTube Videos Create unlisted placeholder videos on YouTube for each week, add those links to Canvas initially, then replace the actual video content by uploading over the placeholders (keeping the same YouTube URL). Practical Recommendation: Given your technical background with Unity, the redirect approach is probably your best bet. It's simple, reliable, and gives you complete control. You could even build a small web interface to manage the redirects if you're handling many classes. The Canvas API approach would be more automated but requires more initial setup.
